## 业务背景

### 问题挑战
某电商平台（日活100万+）面临以下挑战：

1. **营销成本失控**
   - 传统全量发放导致月度预算超支35%
   - 高价值用户获得优惠不足，流失率上升

2. **用户体验差**
   - 低价值用户获得大额优惠但转化率低
   - 高价值用户抱怨"不公平"
   - 用户满意度从78%降至62%

3. **资源浪费**
   - 40%的优惠券未被使用
   - 高价值优惠券被非目标用户获取
   - 营销ROI从3.2x降至2.1x

### 解决方案
引入加权随机系统，实现：
- ✅ 精准资源分配
- ✅ 预算严格控制
- ✅ 个性化用户体验
- ✅ 可审计的公平性

### 第二阶段：系统开发 (Week 3-4)

#### 核心实现
```python
class EcouponSystem:
    def __init__(self, daily_budget=50000):
        self.system = WeightedRandomSystem(
            weight_calculator=LogarithmicWeightCalculator(),
            weight_adjusters=[
                UserFactorAdjuster(),
                BudgetConstraintAdjuster()
            ]
        )
        self.budget_control = BudgetControlledSelector(
            self.system,
            total_budget=daily预算
        )

    def distribute_coupon(self, user):
        # 计算用户因子
        user_factor = self.calculate_user_score(user)

        # 根据用户等级过滤优惠券
        available_discounts = self.filter_by_user_level(
            self.discount_items,
            user.tier
        )

        # 加权随机选择
        result = self.budget_control.select_with_budget(
            available_discounts,
            seed=f"{user.id}_{self.get_week_number()}",
            context={
                'user_factor': user_factor,
                'user_tier': user.tier
            }
        )

        return result
```

#### 关键设计决策
1. **对数权重**: 防止权重差异过大，避免极端情况
2. **确定性种子**: 确保同一用户在短期内获得相同优惠
3. **预算约束**: 严格控制每日营销成本
4. **分层控制**: 不同等级用户可获得的优惠范围不同

## 最佳实践总结

### ✅ 建议做法
1. **从小规模开始**: 先做A/B测试验证效果
2. **数据驱动决策**: 基于实际数据优化权重
3. **实时监控**: 设置预警机制，及时发现问题
4. **用户反馈**: 收集用户反馈，持续优化体验

### ❌ 避免陷阱
1. **权重过于极端**: 1000 vs 1 导致其他选项无法选中
2. **忽略边界情况**: 权重全为0、负权重等
3. **缺乏透明度**: 用户不理解为什么得到某优惠
4. **过度优化**: 追求极致ROI损害用户体验
