# 热点地图系统 v1.1 更新说明

## 📅 更新日期
2026-03-17

## 🎉 新增功能

### 1. 后端 API 服务器
- **技术栈**: Node.js + Express + CORS
- **端口**: 3000
- **数据存储**: 内存存储（26 个模拟热点）
- **功能**: 完整的 REST API 支持

### 2. 智能搜索功能
- **搜索范围**: 标题、城市、地区、作者
- **实时反馈**: 显示搜索结果数量和状态
- **快捷键**: 支持回车键快速搜索
- **UI 优化**: 美观的搜索框设计

### 3. 实时数据刷新
- **一键刷新**: 点击按钮即可获取最新数据
- **加载状态**: 显示刷新进度和状态
- **状态保持**: 刷新后保持当前的筛选条件
- **自动降级**: API 不可用时自动使用模拟数据

### 4. REST API 端点

| 方法 | 端点 | 说明 | 参数 |
|------|------|------|------|
| GET | `/api/health` | 健康检查 | - |
| GET | `/api/hotspots` | 获取热点列表 | platform, type, city, minScore, maxScore, limit, offset |
| GET | `/api/hotspots/:id` | 获取热点详情 | - |
| GET | `/api/hotspots/search` | 搜索热点 | q（搜索关键词） |
| GET | `/api/stats` | 获取统计数据 | - |
| POST | `/api/fetch` | 触发数据抓取 | - |
| POST | `/api/hotspots` | 添加新热点 | hotspot 对象 |
| PUT | `/api/hotspots/:id` | 更新热点 | 更新字段 |
| DELETE | `/api/hotspots/:id` | 删除热点 | - |

## 🏗️ 架构改进

### 前后端分离
- **前端**: 原生 HTML/CSS/JavaScript + Leaflet
- **后端**: Node.js + Express API
- **通信**: RESTful API + JSON
- **降级策略**: API 失败时自动使用模拟数据

### 数据流
```
用户操作 → 前端处理 → API 请求 → 后端处理 → 数据返回 → 前端渲染
     ↓
降级路径: API 请求失败 → 使用模拟数据 → 前端渲染
```

## 📝 开发者笔记

### 技术决策
1. **选择 Node.js + Express**: 简单快速，适合 MVP
2. **内存存储**: 便于开发和测试，后续可升级为数据库
3. **降级策略**: 提高可用性，即使后端失败也能使用
4. **RESTful API**: 标准化接口，易于扩展

### 已知问题
- 模拟数据的时间戳都是当前时间
- 刷新按钮没有自动刷新机制（需要手动点击）
- 搜索结果过多时可能影响性能
