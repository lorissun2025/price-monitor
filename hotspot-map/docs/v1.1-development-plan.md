## 📅 开发阶段

### 阶段 1：后端 API 和真实数据

#### 1.1 API 设计

**端点规划：**
```
GET  /api/hotspots              # 获取所有热点
GET  /api/hotspots/:id          # 获取单个热点详情
GET  /api/hotspots/trending     # 获取趋势热点
GET  /api/hotspots/stats        # 获取统计数据
POST /api/hotspots              # 创建热点（管理端）
PUT  /api/hotspots/:id          # 更新热点
DELETE /api/hotspots/:id        # 删除热点
```

**数据模型：**
```typescript
interface Hotspot {
  id: string;
  title: string;
  description: string;
  type: 'food' | 'travel' | 'event' | 'acg' | 'social';
  platform: 'xiaohongshu' | 'weibo' | 'douyin' | 'bilibili';
  location: {
    name: string;
    lat: number;
    lng: number;
  };
  metrics: {
    views: number;
    likes: number;
    comments: number;
    shares: number;
    influence: number;  // 影响力评分 (0-1)
  };
  trend: {
    history: Array<{
      timestamp: number;
      views: number;
      influence: number;
    }>;
    growth: number;  // 增长率
  };
  createdAt: number;
  updatedAt: number;
}
```

#### 1.3 数据抓取策略

**小红书：**
- 使用网页版 API 或爬虫
- 抓取热门笔记
- 提取：标题、内容、地点、互动数据

**微博：**
- 微博热搜 API
- 热门话题数据
- 提取：话题名称、热度、相关地点

**抖音：**
- 热门视频数据
- 需要破解反爬机制
- 提取：视频标题、位置、播放数据

**B站：**
- 热门视频 API
- 提取：标题、UP主、播放量、弹幕数

#### 1.4 数据存储

**SQLite 表结构：**
```sql
-- 热点表
CREATE TABLE hotspots (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  type TEXT NOT NULL,
  platform TEXT NOT NULL,
  location_name TEXT,
  location_lat REAL,
  location_lng REAL,
  views INTEGER DEFAULT 0,
  likes INTEGER DEFAULT 0,
  comments INTEGER DEFAULT 0,
  shares INTEGER DEFAULT 0,
  influence REAL DEFAULT 0,
  growth REAL DEFAULT 0,
  created_at INTEGER,
  updated_at INTEGER
);

-- 历史数据表（用于趋势分析）
CREATE TABLE hotspot_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  hotspot_id TEXT NOT NULL,
  timestamp INTEGER NOT NULL,
  views INTEGER DEFAULT 0,
  influence REAL DEFAULT 0,
  FOREIGN KEY (hotspot_id) REFERENCES hotspots(id)
);

-- 索引
CREATE INDEX idx_hotspots_type ON hotspots(type);
CREATE INDEX idx_hotspots_platform ON hotspots(platform);
CREATE INDEX idx_hotspots_influence ON hotspots(influence DESC);
CREATE INDEX idx_history_hotspot ON hotspot_history(hotspot_id);
CREATE INDEX idx_history_timestamp ON hotspot_history(timestamp);
```

### 阶段 3：趋势分析功能

#### 3.1 时序数据设计

**数据收集策略：**
- 每小时记录一次热点数据
- 存储：views, likes, comments, influence
- 保留最近 7 天的历史数据

**趋势计算：**
```typescript
// 增长率计算
function calculateGrowth(current: number, previous: number): number {
  if (previous === 0) return 0;
  return ((current - previous) / previous) * 100;
}

// 趋势类型
enum TrendType {
  RISING = 'rising',      // 上升
  STABLE = 'stable',      // 稳定
  FALLING = 'falling',    // 下降
  VOLATILE = 'volatile'   // 波动
}
```

#### 3.2 趋势图表

**使用 Recharts 展示：**
- 折线图：热度随时间变化
- 面积图：影响力变化
- 柱状图：各平台热度对比

#### 3.3 预测模型（基础版）

**简单移动平均预测：**
```typescript
function predictNextValue(history: number[]): number {
  const last5 = history.slice(-5);
  return last5.reduce((a, b) => a + b, 0) / last5.length;
}
```
