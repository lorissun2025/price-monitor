## 🔧 P0 修复（立即执行）

### 修复 #1: 调整超时时间

**当前配置**:
```json
{
  "agents": {
    "defaults": {
      // 缺少 timeoutMs 配置，默认为 600000ms (10分钟)
    }
  }
}
```

**修复后配置**:
```json
{
  "agents": {
    "defaults": {
      "timeoutMs": 150000,  // 150秒 (2.5分钟)
      "timeoutMsFallback": 60000  // 降级模式: 60秒
    }
  }
}
```

**预期效果**:
- 用户等待时间从 10 分钟降到 2.5 分钟
- 快速失败，减少无效等待
- 降级模式下 60 秒超时

**验证命令**:
```bash
# 修改配置后重启 Gateway
openclaw gateway restart

# 查看日志确认新超时时间
grep "timeoutMs" /Users/sunsensen/.openclaw/logs/gateway.err.log | tail -3
```

### 修复 #2: 启用重试机制

**当前配置**:
```json
{
  "agents": {
    "defaults": {
      // 缺少 retry 配置，超时后直接失败
    }
  }
}
```

**修复后配置**:
```json
{
  "agents": {
    "defaults": {
      "retry": {
        "maxAttempts": 2,
        "backoffMs": 1500,
        "exponentialBackoff": true,
        "retryOn": ["timeout", "5xx"],
        "retryOnStatus": [502, 503, 504, 521, 522, 524]
      }
    }
  }
}
```

**预期效果**:
- 临时性超时自动重试
- 最多 2 次重试，指数退避（1.5秒, 3秒）
- 仅对 timeout 和 5xx 错误重试
- 失败率从 100% 降到 < 10%

**验证命令**:
```bash
# 触发一个超时任务，观察是否重试
# 查看日志中的重试记录
grep -i "retry" /Users/sunsensen/.openclaw/logs/gateway.err.log | tail -10
```

```json
{
```

```json
      "memory_search", "memory_get",
      "sessions_spawn", "sessions_yield"

  }
}
```

**预期效果**:
- 消除每 10 分钟一次的 allowlist 警告
- 日志更干净

## 🔧 P1 修复（24小时内）

### 修复 #4: 实现降级执行模式

**问题**: 任务接近超时时没有简化执行方案

**配置方案**:
```json
{
  "agents": {
    "defaults": {
      "degradation": {
        "enabled": true,
        "triggerThreshold": 0.7,  // 超过 70% 超时时间时触发
        "mode": "lightweight",
        "strategies": {
          "reduceContext": true,    // 减少上下文长度
          "limitTools": 3,          // 限制工具调用次数
          "shortenOutput": true,     // 缩短回答长度
          "skipReasoning": true      // 跳过 reasoning 模式
        }
      }
    }
  }
}
```

**降级效果**:
- 上下文: 完整 → 摘要（< 10K tokens）
- 工具调用: 无限制 → 最多 3 次
- 回答长度: 无限制 → < 1000 字
- 模式: reasoning → 普通

**触发条件**: `elapsedTime > 0.7 * timeoutMs` (即 > 105秒)

**验证命令**:
```bash
# 触发一个长任务，观察是否降级
grep -i "degrad\|fallback\|lightweight" /Users/sunsensen/.openclaw/logs/gateway.err.log
```

### 修复 #5: 添加熔断机制

**问题**: 连续超时不触发熔断，继续浪费资源

**配置方案**:
```json
{
  "agents": {
    "defaults": {
      "circuitBreaker": {
        "enabled": true,
        "failureThreshold": 3,        // 3 次连续失败
        "timeoutThreshold": 300000,   // 5 分钟内
        "openDuration": 300000,       // 熔断 5 分钟
        "halfOpenDuration": 60000,    // 半开状态 60 秒
        "lightweightOnly": true       // 熔断期间只允许轻量请求
      }
    }
  }
}
```

**熔断状态**:
1. **Closed (正常)**: 正常处理请求
2. **Open (熔断)**: 拒绝大部分请求，只允许轻量请求（< 30秒）
3. **Half-Open (半开)**: 尝试恢复，如果成功则回到 Closed，否则继续 Open

**预期效果**:
- 防止连续超时浪费资源
- 熔断期间只允许轻量请求
- 5 分钟后自动尝试恢复

**验证命令**:
```bash
# 连续触发 3 次超时，观察熔断
grep -i "circuit\|break\|open\|closed" /Users/sunsensen/.openclaw/logs/gateway.err.log
```

### 修复 #6: 修复 Feishu token 刷新

**当前问题**:
```
[delivery-recovery] Retry failed for delivery xxx: Cannot destructure property 'tenant_access_token' of '(intermediate value)' as it is undefined.
```

**原因**: Feishu API 调用中 `tenant_access_token` 未正确获取

**修复方案**:

**步骤 1**: 检查 Feishu 配置
```bash
# 查看当前 Feishu 配置
cat /Users/sunsensen/.openclaw/openclaw.json | jq '.channels.feishu'
```

**步骤 2**: 修复 token 获取逻辑（需要修改 Feishu 插件代码）
```javascript
// /usr/local/lib/node_modules/openclaw/extensions/feishu/delivery.js
// 确保在请求前获取 token
async function ensureToken() {
  if (!this.tenantAccessToken || isTokenExpired(this.tenantAccessToken)) {
    this.tenantAccessToken = await this.refreshToken();
  }
  return this.tenantAccessToken;
}
```

**步骤 3**: 添加 token 刷新重试
```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "tokenRefresh": {
        "autoRefresh": true,
        "refreshBeforeExpiry": 300  // 提前 5 分钟刷新
      }
    }
  }
}
```

**验证命令**:
```bash
# 发送一条消息到 Feishu，观察是否成功
# 查看 Delivery 日志
grep "delivery" /Users/sunsensen/.openclaw/logs/gateway.err.log | tail -10
```
