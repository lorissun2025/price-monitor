## 1. 产品概述

### 1.1 产品定位

热点地图系统是一个**地理位置化的热点内容可视化平台**，聚合多个主流社交媒体平台的热点内容，通过地图形式直观展示热点分布、影响力、趋势变化，帮助用户快速了解城市/区域内的热点动态。

### 1.2 目标用户

- **营销从业者:** 了解区域热点趋势，制定精准营销策略
- **旅游达人:** 发现热门打卡地点，规划旅行路线
- **美食博主:** 追踪热门餐厅和美食趋势
- **媒体工作者:** 快速获取热点新闻，寻找报道素材
- **普通用户:** 发现身边有趣的地方和活动

### 1.3 核心价值

- 🎯 **一站式聚合** - 无需切换多个 App，一个平台看全热点
- 📍 **地理可视化** - 直观展示热点分布，发现身边新鲜事
- ⚡ **实时更新** - 热点实时追踪，不错过任何趋势
- 📊 **影响力评分** - 多维度分析热点传播度和影响力

## 2. 产品功能

### 2.1 核心功能 (MVP)

#### 2.1.1 多平台数据聚合

**支持平台:**
- 📕 小红书 - 美食、旅游、生活方式
- 📟 微博 - 新闻热点、社会话题
- 🎵 抖音 - 短视频热点、网红打卡地
- 📺️ B站 - ACG、游戏、二次元内容

**数据维度:**
- 内容标题和摘要
- 地理位置（城市、区域、坐标）
- 热度数据（点赞、收藏、播放、讨论数）
- 发布时间
- 作者信息

#### 2.1.2 地图可视化展示

**地图类型:**
- 🇨🇳 中国地图（聚焦国内热点）
- 🌍 世界地图（支持国际热点）

**可视化元素:**
- 🔴 **热点标记** - 不同颜色表示不同影响力等级
- 🌊 **扩散圆环** - 半径表示影响范围，动画表示热度
- 📍 **平台图标** - 直观区分数据来源
- 💬 **信息弹窗** - 点击查看热点详细信息

**交互功能:**
- 缩放、平移地图
- 点击热点查看详情
- 拖拽调整视角

#### 2.1.3 智能筛选系统

**快捷筛选:**
- 📍 全部热点
- 🍜 美食探店
- 🏝 旅游打卡
- 🎪 漫展活动
- 📰 社交热点

**高级筛选:**
- 按平台筛选（多选）
- 按影响力范围筛选
- 按时间范围筛选（今日/本周/本月）
- 按地理位置筛选（城市/省份）

#### 2.1.4 热点影响力评分

**评分维度:**
- 传播度（点赞/播放/收藏数）
- 讨论度（评论/分享/讨论数）
- 时效性（发布时间衰减）
- 持续性（热度保持时长）

**影响力等级:**
- 🔥 极高影响力 (>0.8)
- ⭐ 高影响力 (0.6-0.8)
- ⭐⭐ 中等影响力 (0.4-0.6)
- ⭐⭐ 新兴热点 (<0.4)

#### 2.1.5 热点详情展示

**详细信息:**
- 标题和摘要
- 平台来源
- 影响力评分和等级
- 位置信息（城市、区域）
- 热度数据（各平台特有指标）
- 作者信息
- 发布时间
- 原始链接（跳转到原始内容）

#### 2.1.6 实时统计面板

**统计指标:**
- 当前展示热点数量
- 各平台热点数量
- 影响力分布
- 热点类型分布

### 2.2 增强功能 (v1.1)

#### 2.2.1 后端 API 服务器

- RESTful API 架构
- 数据聚合和缓存
- 自动降级机制

#### 2.2.2 智能搜索功能

- 快捷键支持
- 搜索结果统计

#### 2.2.3 实时数据刷新

- 一键刷新最新数据

#### 2.2.4 个性化推荐

- 基于用户位置推荐附近热点
- 基于历史行为推荐兴趣内容
- 热点订阅和推送

### 2.3 增强功能 (v1.2)

#### 2.3.1 用户位置定位

**功能描述:**
- 使用浏览器 Geolocation API 获取用户位置
- 优雅的定位按钮和错误处理
- 在地图上显示用户位置标记和精度圈
- 位置缓存和复用

**使用场景:**
- 快速查看自己位置附近的热点
- 了解距离，规划出行路线

#### 2.3.2 距离计算和显示

**功能描述:**
- 使用 Haversine 公式计算两点间距离
- 自动计算所有热点到用户的距离
- 距离格式化显示（km/m 自动切换）
- 在热点详情中显示距离信息

**使用场景:**
- 知道热点距离，判断是否值得去
- 比较多个热点的距离，选择最近的

#### 2.3.3 距离筛选功能

**功能描述:**
- 按距离筛选热点（1km/5km/10km/50km）
- 实时筛选更新
- 筛选状态提示

**使用场景:**
- 快速找到附近的热点
- 根据出行时间筛选合适的距离范围

#### 2.3.4 距离排序功能

**功能描述:**
- 按距离从近到远排序热点
- 一键切换排序方式（距离/影响力）
- 排序状态提示

**使用场景:**
- 找到最近的热点
- 规划出行路线（从近到远）

#### 2.3.5 距离统计面板

**功能描述:**
- 显示最近的热点距离
- 显示平均距离
- 统计各距离范围内的热点数量（5km/10km/50km）

**使用场景:**
- 了解整体距离分布
- 快速判断是否有近距离热点

#### 2.3.6 导航链接

**功能描述:**
- 在热点详情中添加导航链接
- 支持高德地图、百度地图导航
- 一键跳转

**使用场景:**
- 查找到满意的热点后，直接导航前往

## 4. 非功能性需求

### 4.1 性能要求

- **页面加载时间:** < 3 秒
- **地图渲染:** < 1 秒
- **热点筛选响应:** < 500ms
- **数据更新频率:** 每 10 分钟刷新一次

### 4.2 可用性要求

- **系统可用性:** 99.5% 以上
- **同时在线用户:** 支持 100+ 并发访问

### 4.3 兼容性要求

- **浏览器:** Chrome、Firefox、Safari、Edge (最新版本)
- **移动端:** 响应式设计，支持手机和平板
- **屏幕分辨率:** 1920x1080 及以上

### 4.4 安全性要求

- 数据传输使用 HTTPS 加密
- API 接口访问频率限制
- 敏感数据脱敏处理
- 用户数据隐私保护

### 4.5 可维护性要求

- 模块化代码架构
- 完善的日志记录
- 清晰的 API 文档
- 自动化测试覆盖率 > 80%

## 5. 技术约束

### 5.1 数据源约束

- **平台 API 限制:** 各平台有 API 调用频率限制，需遵守
- **数据合规性:** 遵守各平台的数据使用条款
- **数据存储:** 本地存储，不上传用户数据

### 5.2 地图服务

- **地图提供商:** Leaflet (开源) + OpenStreetMap / ArcGIS
- **地图瓦片:** 免费瓦片服务，遵守使用条款

### 5.3 部署环境

- **前端:** 静态网站托管 (Vercel / Netlify / GitHub Pages)
- **后端:** Node.js + Express (可选，用于数据聚合)
- **数据库:** SQLite / JSON 文件存储 (轻量级)
