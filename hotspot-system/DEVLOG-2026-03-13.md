## 🚀 使用流程

### 准备工作
```bash
# 1. 登录 Firecrawl
firecrawl login --browser

# 2. 验证状态
firecrawl --status
```

### 启动抓取
```bash
# 3. 启动 API
cd api && npm run dev

# 4. 触发抓取（方法 1）
curl -X POST http://localhost:3000/api/fetch

# 5. 触发抓取（方法 2）
npm run test-scraper

# 6. 查看进度
curl http://localhost:3000/api/fetch/status/fetch_<taskId>
```

### 查看结果
```bash
# 7. 查看统计数据
curl http://localhost:3000/api/stats

# 8. 查看热点数据
curl http://localhost:3000/api/hotspots

# 9. 查看前端
open hotspot-map/index.html
```

## 🎯 技术要点

### 并行抓取
使用 `Promise.allSettled()` 并行抓取四个平台：
- 部分失败不影响其他平台
- 自动记录失败原因
- 完整的错误处理

### 数据转换
```javascript
// Firecrawl 结果 → 热点格式
{
  id: generateHotspotId(platform, index),
  platform: 'xiaohongshu',
  title: result.title,
  type: inferHotspotType(title, description),
  influenceScore: calculateInfluenceScore(metrics),
  location: mockGeoLocation(city),
  metrics: generateMetrics(platform),
  metadata: {...}
}
```

### 影响力评分
```javascript
score = weighted_sum(
  likes, views, readCount,  // 主要指标
  shares, comments, collects // 次要指标
) / max_value
```

### 数据去重
- 按 ID 去重
- 保留最新版本
- 按影响力和时间排序
- 限制总数（默认 100）

## 📝 注意事项

1. **Firecrawl 认证**
   - 需要登录才能使用
   - 每月有免费额度限制
   - 超限需要付费

2. **网络请求**
   - 首次抓取较慢（30-60 秒）
   - 可能触发反爬限制
   - 建议控制频率

3. **数据质量**
   - 当前使用模拟地理位置
   - 部分数据可能不准确
   - 需要人工审核

4. **合规性**
   - 遵守网站 robots.txt
